# 🔧 附着模式命名改进

## 📋 改进说明

根据用户反馈，将附着模式 `WALL_OPENING` 的**显示名称**从"墙洞口"改为"墙体"。

## 💡 改进原因

### 之前：❌ "墙洞口"
- **语义不清**：是"墙上的洞口"还是"在墙上开洞的构件"？
- **容易混淆**：用户可能理解为"洞口本身"而不是"会开洞的构件"
- **不直观**：需要额外解释

### 现在：✅ "墙体"
- **更直观**：表示"附着在墙体上的构件"
- **隐含语义**：附着在墙体上的构件（门、窗）自然会自动开洞
- **更简洁**：用户容易理解，符合直觉

## 🔧 技术实现

### 保持枚举值不变
```java
// AttachmentType.java - 保持不变
public enum AttachmentType {
    WALL_OPENING,  // 技术术语，语义清晰
    // ...
}
```

**优势**：
- ✅ 向后兼容（已保存的数据不受影响）
- ✅ 代码语义明确（`WALL_OPENING` 清楚表示"会在墙上开洞"）

### 只修改显示名称
```java
// ComponentCapturePanel.java
private String getAttachmentModeDisplay() {
    return switch (attachmentMode) {
        case WALL_OPENING -> "墙体"; // 门、窗等会在墙体上开洞的构件
        // ...
    };
}
```
