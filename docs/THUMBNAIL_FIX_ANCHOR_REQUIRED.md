# 🎯 缩略图显示修复 - 需要设置锚点

## 🐛 问题根源

根据日志分析，缩略图无法显示的根本原因是：**锚点未设置！**

```
✗ buildCurrentComponentJson 返回 null
```

## 🔍 技术细节

在 `ComponentTool.java:890` 中有这样的检查：

```java
// v1：Anchor 必须显式选择（不再默认选区 min）
BlockPos anchor = state.anchorWorld;
if (!isInsideSelection(anchor)) return null;
```

**如果锚点（anchor）没有设置或不在选区内，`buildCurrentComponentJson()` 会返回 null！**

这导致：
1. 无法生成构件定义 JSON
2. 无法生成缩略图
3. 面板一直显示"生成中..."

## ✅ 解决方案

### 修复内容

1. **添加防抖机制**
   - 防止每帧都创建新线程
   - 使用 `isGeneratingThumbnail` 标志避免重复生成
   - 线程命名为 "ThumbnailGenerator"

2. **添加详细日志**
   - 检查 client 和 world
   - 检查选区是否存在
   - **检查锚点是否设置**
   - 显示选区和锚点坐标

3. **用户友好的错误提示**
   - 如果锚点未设置，明确提示："请先设置锚点（右键点击方块）"

## 🎯 正确的使用流程

### 步骤 1：激活选区工具

1. 打开 FormaCraft UI（按 `G`）
2. 切换到 **🧰 工具** 面板
3. 点击"选区工具"按钮
4. 框选一些方块（例如 5x5x5 的区域）

### 步骤 2：**设置锚点**（重要！）

1. **右键点击选区内的一个方块** 来设置锚点
2. 你应该会看到一个视觉标记显示锚点位置
3. 锚点通常设置在构件的"原点"或"底部中心"

### 步骤 3：切换到构件拾取面板

1. 切换到 **🎯 构件拾取** 面板
2. 现在应该能看到缩略图预览了！

### 步骤 4：保存构件

1. 填写构件名称
2. 点击"保存构件"按钮
3. 自动跳转到构件库，显示刚保存的构件

## 🛠️ 设置锚点的最佳实践

### 对于不同类型的构件

1. **窗户/门**
   - 锚点设置在底部中心
   - 朝向设置为"向外"

2. **柱子**
   - 锚点设置在底部中心
   - 可以任意旋转

3. **装饰物**
   - 锚点设置在附着点
   - 例如：墙面装饰的锚点在墙上

4. **屋顶**
   - 锚点设置在底部中心
   - 朝向与建筑主朝向一致

### 视觉标记

锚点设置后，应该能看到：
- 一个高亮的方块或标记
- 在构件拾取面板中显示锚点坐标
- 锚点的朝向箭头（如果适用）

## 🔧 故障排除

### 问题：右键点击方块没有设置锚点

**可能原因**：
1. 选区工具没有激活
2. 点击的方块不在选区内
3. UI 拦截了右键事件

**解决方案**：
1. 确保选区工具已激活（工具面板中应该高亮显示）
2. 点击选区边界内的方块
3. 如果 UI 打开，确保鼠标不在面板区域内

### 问题：设置了锚点但仍然返回 null

**可能原因**：
1. 锚点不在选区内
2. 选区已经改变但锚点没有更新

**解决方案**：
1. 检查日志中的选区和锚点坐标
2. 确保锚点坐标在选区范围内
3. 重新框选并重新设置锚点

### 设置锚点的方法

目前锚点是通过右键点击方块设置的。在 `ComponentTool` 中应该有类似的逻辑：

```java
// 当用户右键点击方块时
public void setAnchor(BlockPos pos) {
    if (isInsideSelection(pos)) {
        state.anchorWorld = pos;
    }
}
```
