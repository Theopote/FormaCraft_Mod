## 📋 增强内容

### 1. ✅ 语义含义推断（`inferSemanticMeaning`）

**功能**：从标签名称推断建筑语义含义

**支持的标签（20+）**：

| 标签关键词 | 语义含义 |
|-----------|---------|
| 入口 / entrance | 建筑主入口位置，需要门、台阶、装饰性元素，通常是建筑的焦点 |
| 庭院 / courtyard / 中庭 | 开放庭院空间，不覆盖屋顶，较低高度，通常作为建筑内部的开放区域 |
| 住宅 / residential / 居室 | 居住空间，需要窗户、门、合理的空间布局，体现居住功能 |
| 商业 / commercial / 商店 / 商铺 | 商业空间，需要展示窗口、入口、招牌位置，面向街道 |
| 仓库 / warehouse / 库房 | 存储空间，较少窗户，注重实用性，通常较大空间 |
| 办公 / office / 工作 | 办公空间，需要窗户、合理的空间分割，功能性优先 |
| 厨房 / kitchen | 厨房区域，需要功能性布局，考虑通风和实用 |
| 卧室 / bedroom / 房间 | 卧室或房间，需要窗户、门、私密性考虑 |
| 大厅 / hall / 厅堂 | 大厅空间，通常是开放、高大的空间，作为主要活动区域 |
| 走廊 / corridor / 过道 | 连接通道，需要连接不同区域，考虑通行便利 |
| 楼梯 / stair / 阶梯 | 楼梯位置，需要连接不同楼层，考虑垂直交通 |
| 阳台 / balcony / 露台 | 阳台空间，向外突出，需要栏杆、视野考虑 |
| 花园 / garden / 园林 | 花园区域，开放空间，可以有装饰性建筑元素 |
| 广场 / plaza / 空地 | 开放广场，不建造建筑，或只有装饰性建筑元素 |
| 停车场 / parking | 停车区域，通常平坦，可能有简单的结构 |
| 防御 / defensive / 守卫 | 防御性区域，需要围墙、瞭望塔等防御元素 |
| 神圣 / sacred / 宗教 / temple / 庙 | 宗教或神圣空间，需要装饰性元素、对称布局、庄重感 |
| 公共 / public | 公共空间，需要开放、宽敞、便于集会 |
| 私人 / private | 私密空间，需要围合、较少窗户或遮挡，保护隐私 |

**默认行为**：如果标签不在预定义列表中，会根据标签名称的字面意思进行推测。

### 2. ✅ 设计指导推断（`inferDesignGuidance`）

**功能**：从标签名称推断具体的设计指导

**示例**：

- **入口标签**：
  - 指导：在此区域必须包含门（可能是主门），添加台阶、装饰性元素（如门楣、门柱），使其成为建筑的视觉焦点

- **庭院标签**：
  - 指导：此区域应该保持开放，不覆盖屋顶，高度较低，可以包含装饰性围墙、花坛、路径等元素

- **住宅标签**：
  - 指导：在此区域生成居住建筑，包含窗户、门、合理的房间布局，使用居住建筑常见的材料和装饰

- **商业标签**：
  - 指导：在此区域生成商业建筑，包含展示窗口、宽敞入口、招牌位置，面向街道或主要通道

**每个标签都有明确的设计指导**，告诉 AI 应该在该区域生成什么类型的建筑元素。

## 🎯 使用示例

### 示例 1：入口 + 庭院布局

**用户操作**：
1. 使用语义标注工具标注"入口"区域
2. 使用语义标注工具标注"庭院"区域
3. 输入："生成一个中式建筑"

**AI 行为**：
- 在"入口"区域生成：门、台阶、门楣、门柱等装饰性元素
- 在"庭院"区域生成：开放的庭院空间，不覆盖屋顶，可能包含装饰性围墙、花坛
- 根据标签的 `range` 值，确保周边区域也体现相应的建筑特征

### 示例 2：功能分区

**用户操作**：
1. 标注"商业"区域（range=10）
2. 标注"住宅"区域（range=10）
3. 输入："生成一个前店后宅的建筑"

**AI 行为**：
- 在"商业"区域生成：展示窗口、宽敞入口、招牌位置，面向街道
- 在"住宅"区域生成：居住建筑，包含窗户、门、合理的房间布局
- 在 range 范围内保持一致的建筑特征

```
区域语义标注（CRITICAL - MUST USE FOR GUIDED GENERATION）：
- 以下标签把自然语言意图与空间区域绑定，你必须严格遵守并利用这些标签指导建筑生成
- 每个标签都有明确的建筑语义含义，你需要根据标签名称来理解应该在该区域生成什么类型的建筑元素
- range 字段表示标签的作用范围（方块半径），影响周边区域的建筑设计决策
- 多个标签可以组合使用，形成复杂的建筑布局

  Area 1 - 标签: "入口"
  - 语义含义: 建筑主入口位置，需要门、台阶、装饰性元素，通常是建筑的焦点

  - 作用范围: 16 格（该范围内的建筑应该体现此标签的特征）
  - 高度范围: Y=64~84
  - 区域边界（多边形顶点，X,Z 坐标）：
    - (100,200)
    - (105,200)
    ...

使用规则：
- 当生成建筑的组件时，检查该组件的位置是否落在某个标签区域内
- 如果落在标签区域内，必须根据标签的语义含义来设计该组件
- 例如：标签为"入口"的区域应该包含门、台阶、装饰性元素；标签为"庭院"的区域应该是开放的、没有屋顶的空间
- 标签的 range 值告诉你该标签影响的周边范围，在该范围内应该保持一致的建筑特征
```

**优势**：
- ✅ AI 明确知道"入口"的含义和应该生成什么
- ✅ 有具体的设计指导
- ✅ 解释了 `range` 的作用
